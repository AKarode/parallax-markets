    ```python
    def recalibrate_probability(
        raw_prob: float,
        model_id: str,
        conn: duckdb.DuckDBPyConnection,
        min_signals: int = 10,
        max_offset: float = 0.15,
    ) -> tuple[float, float]:
        """Apply bucket-based recalibration. Returns (calibrated, raw).
        
        Only activates when min_signals resolved signals exist for the model.
        Offset per bucket capped at max_offset to prevent oscillation.
        """
    ```

    Implementation:
    1. Count resolved signals for model_id: `SELECT COUNT(*) FROM signal_ledger WHERE model_id = ? AND model_was_correct IS NOT NULL`
    2. If count < min_signals, return (raw_prob, raw_prob)
    3. Call `calibration_curve(conn, model_id=model_id)` to get per-model buckets
    4. Find matching bucket for raw_prob (bucket ranges: 0.0-0.2, 0.2-0.4, 0.4-0.6, 0.6-0.8, 0.8-1.0)
    5. Compute offset = avg_predicted - actual_rate (positive means overestimating)
    6. Cap offset: `offset = max(-max_offset, min(max_offset, offset))`
    7. Calibrated = raw_prob - offset
    8. Clamp to [0.0, 1.0]: `calibrated = max(0.0, min(1.0, calibrated))`
    9. Return (calibrated, raw_prob)

    Helper: `_bucket_for_prob(prob: float) -> str` returns bucket string like "0.6-0.8".
