# Quick Task 260409-88f: Truth Social Ingestion Module

**Status:** Complete
**Date:** 2026-04-09
**Commits:** cefc9a4, 936204f, e6d9466

## What was built

- **`backend/src/parallax/ingestion/truth_social.py`** — New ingestion module that fetches public posts from Truth Social via the `truthbrush` library. Monitors `@realDonaldTrump` by default, filters for Iran/Hormuz-relevant content using keyword matching, and converts posts to `NewsEvent` objects following the same pattern as `google_news.py` and `gdelt_doc.py`.
- **`backend/tests/test_truth_social.py`** — 18 unit tests covering keyword filtering, NewsEvent conversion, age filtering, dedup via seen_hashes, HTML stripping, graceful error handling, and sort order.
- **`backend/src/parallax/cli/brief.py`** — Wired `fetch_truth_social()` as a third concurrent source in `_fetch_gdelt_events()` alongside Google News and GDELT, with full error isolation via `return_exceptions=True`.
- **`backend/pyproject.toml`** — Added `truthbrush>=0.2` dependency.

## Key decisions

- Used `asyncio.to_thread()` to wrap truthbrush's synchronous API without blocking the event loop
- Iran-topic keyword filter keeps only relevant posts (prevents noise from unrelated Trump posts)
- Graceful degradation: if Truth Social fetch fails, Google News and GDELT still work
- Version constraint `>=0.2` (not `>=0.5` as initially planned — truthbrush is at v0.2.5)
