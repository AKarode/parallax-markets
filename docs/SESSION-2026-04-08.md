### 4. Key Design Decisions

- **Two Kalshi endpoints**: Production API (`api.elections.kalshi.com`) for real market prices, Demo API (`demo-api.kalshi.co`) for paper trading. Demo doesn't have geopolitical markets.
- **Dynamic ticker mapping**: Predictions don't hardcode Kalshi tickers. The brief pipeline maps model_id → best matching market at runtime based on available markets.
- **Oil price translation**: Model predicts direction (increase/decrease) → translated to binary probability for WTI price target markets (e.g., "Will WTI hit $120?").
- **3 Sonnet LLM calls per brief**: ~$0.02/run, well under $20/day cap.
- **Google News RSS as primary news source**: Free, no auth, 5-15 min latency. GDELT DOC API as supplementary (gets rate-limited at ~1 req/sec).

## Setbacks and Fixes

### 1. Sync vs Async Anthropic Client
**Problem:** Code used `await self._client.messages.create()` but initialized `anthropic.Anthropic()` (sync client).
**Fix:** Changed to `anthropic.AsyncAnthropic()` in `cli/brief.py:_init_anthropic()`.
**Lesson:** Always use `AsyncAnthropic` when calling from async functions.

### 2. Kalshi Demo vs Production Markets
**Problem:** Demo sandbox (`demo-api.kalshi.co`) only has sports betting and crypto markets — no Iran/Hormuz/oil geopolitical markets.
**Fix:** Use production API (`api.elections.kalshi.com`) for reading market prices, demo for paper trading.
**Lesson:** The demo sandbox mirrors API behavior but NOT market catalog. Always test market discovery on production.

### 3. Kalshi API v2 Field Names
**Problem:** Code expected `yes_bid`, `no_bid`, `volume` fields. Kalshi v2 API returns `yes_bid_dollars`, `no_bid_dollars`, `volume_fp` (float values, not cents).
**Fix:** Updated `kalshi.py:get_market_price()` to use `_dollars` suffix fields.
**Lesson:** Kalshi API docs may not match exactly. Inspect actual response keys via `print(market.keys())`.

### 4. Unfiltered Market Fetch
**Problem:** Initial implementation fetched markets without filter, returning random markets (NYC temperature, NBA stats).
**Fix:** Changed to fetch by specific event tickers (`KXUSAIRANAGREEMENT`, `KXWTIMAX`, `KXCLOSEHORMUZ`, etc.) discovered via paginated event search.
**Lesson:** Kalshi `/markets` endpoint returns by recency, not relevance. Use `event_ticker` parameter to filter.

### 5. Hardcoded Stale Tickers in Prediction Models
**Problem:** Each prediction model hardcoded a `kalshi_ticker` (e.g., `"KXOIL"`, `"KXIRANCEASEFIRE"`) that didn't match any real market.
**Fix:** Removed hardcoded tickers, added `_map_predictions_to_markets()` function in brief.py that dynamically maps model_id to best matching market based on available markets.
**Lesson:** Market tickers change. Never hardcode — always discover at runtime.

### 6. GDELT DOC API Rate Limiting
**Problem:** GDELT DOC API returns 429 when queries are too rapid.
**Current state:** Code has 1-second delays between queries but still gets rate-limited during heavy use.
**Mitigation:** Google News RSS is the primary source; GDELT DOC is supplementary. Errors are caught and logged, not fatal.

### 7. Empty GDELT Events (BigQuery Dependency)
**Problem:** Original `_fetch_gdelt_events()` required BigQuery credentials, returning empty events and causing generic/low-confidence predictions.
**Fix:** Replaced with Google News RSS + GDELT DOC API pollers (both free, no auth needed).

### 8. Env Vars Not Shared Between Processes
**Problem:** User set env vars in their terminal but Claude Code runs in a separate shell process.
**Fix:** User created `/tmp/parallax-env.sh` that Claude Code sources before running commands.
**Lesson:** Use `! export VAR=value` in Claude Code to set in user's terminal, or create a sourceable env file.
