## Setbacks and Fixes

### 1. DuckDB File Not Found
**Problem:** Dashboard showed "Cannot connect to DuckDB" — file at `data/parallax.duckdb` didn't exist.
**Root cause:** CLI defaults to `:memory:` (no persistence). Dashboard expects a file.
**Fix:** Run pipeline with `DUCKDB_PATH=data/parallax.duckdb` to create the file.
**Lesson:** Always set `DUCKDB_PATH` when running the pipeline if you want data to persist for the dashboard.

### 2. PyArrow Broken in Anaconda
**Problem:** `ImportError: dlopen pyarrow/lib.cpython-311-darwin.so — Library not loaded: libarrow.1700.dylib`
**Root cause:** Stale pyarrow 17.0.0 in anaconda had broken dylib references.
**Fix:** `pip install pyarrow --force-reinstall` (upgraded to 23.0.1).
**Lesson:** Anaconda environments can have stale native libraries; force-reinstall when dylib errors appear.

### 3. Plotly 5.10 Doesn't Support 8-Digit Hex Colors
**Problem:** `ValueError: Invalid value '#22c55e20' for color property`
**Root cause:** Plotly 5.10 doesn't accept 8-digit hex with alpha channel.
**Fix:** Changed to `rgba(34,197,94,0.12)` format.
**Lesson:** Use `rgba()` not hex+alpha for Plotly colors.

### 4. Plotly `update_layout()` Duplicate Kwargs
**Problem:** `TypeError: got multiple values for keyword argument 'yaxis'`
**Root cause:** Shared `PLOTLY_LAYOUT` dict had `yaxis`/`xaxis`, then per-chart calls passed them again via `**kwargs`.
**Fix:** Removed axis styles from shared dict; created `_AX` and `_AX_Z` dicts applied per-chart.
**Lesson:** Don't put axis config in shared layout dicts if charts override them.

### 5. Streamlit Iframe Clipping in Playwright
**Problem:** `fullPage: true` screenshots only captured the top portion; charts below the fold weren't visible.
**Root cause:** Streamlit renders inside an iframe; Playwright's fullPage captures outer document height, not iframe content height.
**Fix:** Not fixable from Playwright — user takes screenshots directly in browser.
**Lesson:** Streamlit + Playwright fullPage screenshots are unreliable for long pages.

### 6. ANTHROPIC_API_KEY Not Persisted
**Problem:** User set the key yesterday but it wasn't in the shell today.
**Root cause:** Exported in a terminal session, not added to `~/.zshrc`. Closing terminal loses it.
**Fix:** Created `backend/.env` file (gitignored) for all API keys.
**Lesson:** Always use `.env` files, not ephemeral exports.
